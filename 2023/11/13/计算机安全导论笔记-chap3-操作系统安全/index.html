计算机安全导论笔记 - chap3 - 操作系统安全

操作系统概念

计算机系统

计算机系统：包括计算机硬件系统和计算机软件系统。

计算机硬件由五大部件组成：运算器、控制器、存储器、输入设备和输出设备。

操作系统概念

操作系统：提供计算机用户和该计算机硬件之间的接口，管理应用程序访问计算机资源的方式。
主要资源：磁盘、CPU、主存储器、输入输出设备和网络接口等。
它是用户、应用程序和计算机硬件之间的“粘合剂”，允许应用程序开发人员只编写程序，而无需处理底层细节。

内核

内核（Kernel）：操作系统的核心组件，处理对底层硬件资源的管理。
底层资源：内存、CPU、输入输出设备(键盘、鼠标、显示器)。

输入输出

设备驱动程序（Device driver）：封装了如何与该设备进行交互的细节，完成硬件设备电子信号与操作系统及软件的高级编程语言之间的互相翻译。

应用程序接口（API）：设备驱动程序提供给应用程序的接口，允许程序与设备进行交互，交互细节由操作系统完成。

系统调用

系统调用（System call）：应用程序不能直接与底层硬件进行通信，需要将任务委托给内核。
系统调用通常是一组程序的集合，也称作库（library）。
库函数提供操作接口，允许应用程序使用一系列预定义的 API 与内核进行通信（e.g. 文件 I/O（打开、关闭、读、写），运行应用程序（exec）等）。

许多系统采用软中断（software interrupt）的方式实现系统调用。
软中断流程：1. 应用程序发送请求；2. 处理器停止当前的执行流；3. 处理器切换到内核模式，执行中断要求的处理程序。
进入内核空间涉及直接与硬件交互，所以操作系统限制应用程序与内核交互方法和手段，以提高安全性和正确性。

进程

进程是正在执行的程序的一个实例，程序的实际内容存储在永久性存储器上，如磁盘，SSD 等
程序执行时，一段程序必须加载指随机存取存储器，即内存（RAM）。
一个程序的多个拷贝可以作为不同的进程来执行。
多个进程能够同时运行依赖于对 CPU 的时间切片（time slicing）。

进程树

进程树（Process tree）：用户创建一个新进程时，内核将其视为已有进程请求建立一个新进程。此过程中，已有进程称为父进程，被创建的新进程为子进程。这棵树的根是进程 init 。

进程 ID

进程 ID （PID）：一个给定计算机上运行的每个进程都由唯一的非负整数来标识。

信号机制

进程间通信（Inter-process communication）：为了管理共享资源，进程间必须进行通信。

  1. 进程通信方法 1：使用读写文件来传递信息。
    缺点：
    1.文件处理需要从磁盘读写文件，与RAM比慢很多。
    2.如果两个进程间希望进行私密通信，使用文件传递消息会比较麻烦。

  2. 进程通信方法 2：共享同一块物理内存区，通过共享的 RAM 来传递消息。
    只要内核能得当地管理共享内存空间，这种方法就会快速高效。

  3. 进程通信方法 3：管道（pipe）和套接字（socket），提供一个进程到另一个进程的隧道。

  4. 进程通信方法 4：信号（signal），进程间发送异步消息。本质上是一个进程发送给另一个进程的通知。
    例如 Linux 系统中，命令窗口输入 Ctrl + C 可以向进程发送 INT 信号，终止该进程。

RPC & Daemons

远程过程调用（Remote procedure call, RPC）：Windows 系统下类似于信号机制的方法。本质上允许一个进程调用另一个进程中的子例程（e.g. Windows 使用内核级的 TerminateProcess() 来杀死进程，任何进程可以调用它）。

Linux 系统中称后台进程为守护进程（Daemons）。守护进程运行权限高于任何用户，并在登录会话结束前一直存在。
守护进程示例：Web 服务器、远程登录、打印服务器进程等。
Windows 下与守护进程类似的称为服务（service）。

文件系统

文件系统（File system）：如何组织计算机的外部、非易失性存储器的一种抽象。

文件权限（File permission）：操作系统安全的一个主要问题是如何界定哪些用户可以访问哪些资源，也就是说，哪那个用户可以读取文件、写入数据和执行程序。这一概念封装在文件权限之中，它的具体实现取决于操作系统。磁盘上的每类资源，包括数据文件和程序，都有一个权限与之相关联。操作系统会检查文件权限，以确认用户(组)是否可以读、写、执行一个特定的文件。
共有三类权限：

  1. owner类，它决定了文件创建者的权限。
  2. group类，它决定了相同组中的用户的文件权限。
  3. others类，它决定了既不是所有者的文件，也不是相同组中用户的文件权限。

每一类都有一系列位，用于确定具有的权限。第一位是 read 位，其允许用户读取文件。第二位是 write 位，其允许用户改变文件的内容。最后还有一位 excute 位，其允许用户运行程序或脚本。

虚拟内存

如果每个进程的地址空间都是一个连续的内存块，那么对于使用了大型数组等的程序则需要很大块的连续地址空间，这样内存使用效率会很低。如果进程所需的连续地址空间总量超出了计算机的内存总量，则获取连续内存区是不可能的。

虚拟内存概念的提出解决了这个问题。每个进程都有一个虚拟地址空间，虚拟内存系统将每个虚拟地址映射为实际的内存地址。当访问虚拟地址时，内存管理单元（MMU）硬件组件会查找虚拟地址映射的实际内存地址。对于进程来说，看到的虚拟地址空间是连续的，但实际内存地址空间是不连续的。

虚拟内存另一个优点是，允许正在执行进程的总地址空间大小大于计算机的实际主存储器大小。主要原因是虚拟内存系统可以使用外部驱动器的一部分来存放执行进程不常使用的内存块。

缺点：虚拟内存访问硬盘驱动器要慢于访问 RAM（一般情况下访问硬盘比访问主存慢 10000 倍）。
